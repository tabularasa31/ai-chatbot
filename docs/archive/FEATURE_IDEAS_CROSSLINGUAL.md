# FI-028 Cross-lingual Retrieval

## Проблема

Платформа мультитенантная. Клиенты будут загружать документы на разных языках (RU, EN, и др.). Вопросы пользователей могут быть на любом языке — не обязательно совпадающем с языком документов.

Текущая модель `text-embedding-3-small` неплохо работает с несколькими языками, но качество cross-lingual retrieval (вопрос на EN, документы на RU) может быть хуже, чем monolingual.

## Варианты решения

### Вариант 1: Сменить модель эмбеддингов
- `text-embedding-3-large` — лучше для мультиязычных сценариев.
- `multilingual-e5-large` (open-source) — специально обучена на cross-lingual retrieval.
- `cohere-embed-multilingual-v3` — сильная мультиязычная модель.

### Вариант 2: Translate-then-retrieve
- Перевести вопрос пользователя на язык документов перед поиском.
- Использовать GPT или отдельный translation API.
- Минус: дополнительная latency и стоимость.

### Вариант 3: Hybrid retrieval + reranking
- Использовать multilingual reranker (Cohere Rerank multilingual).
- Сначала широкий retrieval, потом reranking с учётом языка.

## Когда это становится важным
- Когда появятся клиенты с документами на EN и пользователями на RU (или наоборот).
- Когда качество cross-lingual ответов станет заметной проблемой по метрикам.

## Приоритет
P3 — важно для масштабирования на западный рынок, но не блокирует MVP.
