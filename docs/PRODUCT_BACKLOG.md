## RICE-приоритизация (2026-03-18)

RICE = (Reach × Impact × Confidence) / Effort

| Метрика | Шкала |
|---------|-------|
| Reach | 1–10 (кол-во затронутых пользователей/клиентов) |
| Impact | 1=мало, 3=средне, 5=высокая |
| Confidence | % уверенности в оценках |
| Effort | дни разработки |

## Рекомендуемый порядок реализации

### 🔴 Сейчас (эта неделя)

#### [FI-005] Приветственное сообщение в виджете
**RICE: 1440** — лидер

**Что:**
- В настройках клиента хранится строка `greeting_message`.
- При открытии виджета бот показывает это сообщение первым (роль assistant).
- Если не задано — стандартный шаблон ("Привет, я бот поддержки...").

**Ценность:**
- Каждый конечный пользователь виджета видит это сразу.
- Устраняет "страх пустого чата".
- Пользователь сразу понимает, с чем может помочь бот.

**Effort:** 1 день
**Files:** `Client` model, dashboard Settings, `embed.js`

---

#### [FI-007] Per-client system prompt
**RICE: 1020**

**Что:**
- Клиент задаёт текстовое описание своего бота в дашборде:
  - кто он (поддержка CDNvideo / SaaS / API),
  - какой тон/язык использовать,
  - что можно / нельзя отвечать.
- В `build_rag_prompt()` system prompt подмешивается сверху.

**Ценность:**
- Без этого все боты одинаковые.
- Клиент получает контроль над поведением бота без нашего участия.
- Уменьшает "I don't have information about this" — правильный контекст помогает.

**Effort:** 2 дня
**Files:** `Client` model + migration, dashboard Settings page, `chat/service.py`

### 🟠 P2 (западный рынок)

#### [FI-027] Ticketing systems integration (Zendesk, Intercom, Freshdesk)

**RICE:** ~600 (высокий Impact для западной аудитории, средний Reach сейчас)

**Идея:**
Вместо ручного экспорта тикетов — прямая интеграция с тикетными системами.

**Три уровня реализации:**

1. **Импорт тикетов как база знаний (читать)**
   - Клиент подключает Zendesk/Intercom через OAuth или API key.
   - Мы автоматически тянем закрытые тикеты с ответами саппорта.
   - Превращаем в эмбеддинги → бот сразу знает историю поддержки.
   - Синхронизация периодически (раз в день/неделю).
   - **УТП:** "Подключи Zendesk — бот обучается на всей истории."

2. **Escalation (писать)**
   - Если бот не знает → автоматически создаёт тикет в Zendesk.
   - Пользователь видит: "Ваш вопрос передан в поддержку."
   - Замыкает петлю: бот → человек → новый тикет в базе знаний.

3. **Live handoff**
   - Плавный переход к живому агенту через Intercom/Zendesk chat.

**Приоритет интеграций:**
- Zendesk (самый популярный в B2B SaaS)
- Intercom
- Freshdesk
- HubSpot

**Ценность для позиционирования:**
- Сейчас: "Загрузи доки → получи бота."
- С интеграцией: "Подключи Zendesk → бот знает всё и передаёт нерешённые вопросы обратно."

**Effort:** 5–8 дней (Level 1), 10–15 дней (Level 1+2)
