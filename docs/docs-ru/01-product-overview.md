## Chat9 — обзор продукта

### Что такое Chat9

Chat9 — это **мультиарендная SaaS‑платформа**, которая позволяет бизнесам встроить на свой сайт AI‑чат‑бота для поддержки пользователей.

Каждый клиент (арендатор) получает:
- собственный изолированный аккаунт (tenant);
- возможность загрузить свою документацию и знания;
- подключение собственного OpenAI API‑ключа;
- готовый виджет чата, который встраивается на сайт одной строкой.

Бот отвечает на вопросы пользователей с помощью **RAG‑подхода (Retrieval‑Augmented Generation)**: сначала ищет релевантные фрагменты в базе знаний клиента, затем формирует ответ при помощи LLM на основе найденного контекста.

### Основные пользовательские роли

- **Владелец аккаунта / админ клиента**
  - Регистрируется в системе, подтверждает email.
  - Загружает документы и настраивает источники знаний.
  - Подключает свой OpenAI API‑ключ.
  - Встраивает виджет на сайт и контролирует его поведение.
  - Следит за качеством ответов, управляет эскалациями и уровнем детализации.

- **Операторы поддержки клиента**
  - Работают с тикетами эскалаций (L2): видят, где бот не справился, и берут запрос в работу.
  - Просматривают историю диалогов, оставляют обратную связь по ответам (good/bad).

- **Сейлз / аккаунт‑менеджеры (со стороны нашей компании)**
  - Объясняют клиентам, что умеет платформа.
  - Проводят демо, помогают сформировать MVP‑сценарии и запустить пилот.

- **Саппорт (нашей платформы)**
  - Помогает клиентам при технических проблемах: не индексируются документы, виджет не отвечает, ошибки авторизации и т. п.
  - Диагностирует, на каком уровне проблема: фронтенд, бекенд, база знаний, OpenAI, почта и т. д.

- **Конечные пользователи (посетители сайта клиента)**
  - Общаются с виджетом на сайте клиента.
  - Получают ответы по продукту клиента и могут быть эскалированы к живому оператору.

### Какую проблему решает Chat9

Для бизнеса:
- Сократить нагрузку на поддержку за счёт автоматизации типовых вопросов.
- Обеспечить быстрые и консистентные ответы 24/7.
- Сделать использование документации и базы знаний «живой» — не через поиск по странице, а через диалог.
- Быстро запускать AI‑поддержку без собственной ML‑команды.

Для конечного пользователя:
- Получать понятные ответы в естественном языке.
- Не разбираться в структуре документации — бот сам найдёт нужные разделы.
- В сложных случаях быстро перейти к человеку с уже подготовленным контекстом диалога.

### Ключевые сценарии использования

1. **Поддержка SaaS‑продукта**
   - Клиент загружает техническую документацию, FAQ, гайды по настройке.
   - Бот отвечает на вопросы по функционалу, тарифам, интеграциям.
   - Сложные или редкие вопросы эскалируются человеку.

2. **Онбординг новых пользователей**
   - Виджет выступает «виртуальным Customer Success»: помогает сделать первые шаги, объясняет ключевые сценарии продукта.

3. **Внутренняя служба поддержки**
   - Использование Chat9 для внутренних команд (IT‑саппорт, HR, корпоративные политики и процессы).

### Высокоуровневый жизненный цикл клиента

1. **Регистрация и вход**
   - Пользователь создаёт аккаунт, подтверждает email, входит в дашборд.

2. **Базовая настройка**
   - Создаётся tenant‑клиент.
   - Генерируется публичный идентификатор клиента (используется в виджете).
   - Пользователь добавляет и сохраняет OpenAI API‑ключ.

3. **Создание базы знаний**
   - Загрузка файлов (PDF, Markdown, Swagger/OpenAPI).
   - Настройка источников по URL‑адресам документации.
   - Запуск процесса индексации и эмбеддингов.
   - Проверка «здоровья» документов и устранение предупреждений.

4. **Встраивание виджета**
   - Пользователь копирует готовый сниппет и вставляет его в код сайта.
   - Виджет подключается к соответствующему клиенту по публичному идентификатору.

5. **Эксплуатация и улучшение**
   - Клиент следит за качеством ответов (фидбек, bad answers).
   - Регулирует уровень раскрытия информации в ответах (Response controls).
   - Обрабатывает эскалации, дорабатывает документацию и FAQ.

### Чем Chat9 отличается от «простого» ChatGPT‑бота

- **Своя база знаний**: бот обучен не «на всём интернете», а на документации конкретного клиента.
- **Изоляция арендаторов**: данные разных клиентов полностью разделены.
- **Наблюдаемость и контроль**: есть логи, метрики, оценки качества поиска, политики против противоречий в источниках.
- **Инструменты для бизнеса**: управление уровнем детализации ответов, эскалации, KYC/идентификация пользователей виджета, ограничения и rate limiting.
- **Производственный уровень**: продуманная работа с PII, защита от prompt‑injection, бэкенд на PostgreSQL с pgvector, гибридный поиск, CI/CD, прод‑среда.
