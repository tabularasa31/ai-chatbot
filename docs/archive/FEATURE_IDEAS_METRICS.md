# Admin Product Metrics (MVP) — FI-014

**Идея:** Панель метрик для администратора/продакта, чтобы видеть здоровье платформы и отдельных клиентов.

## MVP-метрики (первая версия)

По платформе (summary):
- `total_users` — количество зарегистрированных пользователей
- `total_clients` — количество клиентов
- `active_clients` — клиенты с хотя бы одним документом и хотя бы одной чат-сессией

По клиентам (таблица):
- Client name / id
- `users_count` — сколько пользователей у клиента
- `documents_count` — сколько документов загружено
- `embedded_documents_count` — сколько документов имеют embeddings
- `chat_sessions_count` — сколько сессий чата
- `messages_user_count` / `messages_assistant_count`
- `tokens_used_chat` — суммарно использованных токенов в чатах
- `has_openai_key` — настроен ли OpenAI ключ

По времени (простая сводка по последним N дням):
- новые клиенты в день
- загруженные документы в день
- чат-сессии в день

## Реализация (MVP)

Backend (admin endpoints):
- `GET /admin/metrics/summary` — одна структура с числами по платформе
- `GET /admin/metrics/clients` — список клиентов с колонками выше

Фильтрация:
- пока без сложных фильтров; позже можно добавить `?since=` и пагинацию.

Без отдельной таблицы агрегатов — считаем по существующим моделям (User, Client, Document, Chat, Message) через `GROUP BY`.

Frontend (admin dashboard):
- Страница `/admin/metrics` (доступна только роли admin):
  - сверху 3–4 "плитки":
    - Всего клиентов / активных клиентов
    - Всего документов
    - Всего чат-сессий
    - Токены (примерная оценка)
  - ниже таблица по клиентам:
    - колонки как в списке выше

Роль admin:
- отдельное поле/флаг в User (`is_admin`)
- эндпоинты `/admin/...` доступны только при `is_admin=True`

**Приоритет:** P2 — после внедрения 👍/👎 и Review bad answers, чтобы смотреть качество не только локально, но и на уровне платформы.
