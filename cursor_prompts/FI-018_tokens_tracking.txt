FEATURE:
FI-018 Track tokens used per chat message and aggregate in admin metrics.

GOAL:
Currently `tokens_used` is returned in `/chat` response but NOT saved to DB.
Admin metrics always shows tokens = 0.

We want to:
- Save `tokens_used` on each `Chat` row (total tokens per session message exchange),
- Aggregate it in `/admin/metrics/summary` and `/admin/metrics/clients`.

----------------------------------------------------------------------
1) Add tokens_used field to Chat model
----------------------------------------------------------------------

File: `backend/models.py`

Find the `Chat` model. Add:

```python
class Chat(Base):
    ...
    tokens_used: Mapped[int] = mapped_column(
        Integer,
        nullable=False,
        server_default="0",
    )
```

Note: this stores total tokens used in the most recent message exchange for
this session (user message + assistant response).

----------------------------------------------------------------------
2) Alembic migration
----------------------------------------------------------------------

Create a new migration file in `alembic/versions/`:

```python
"""add tokens_used to chat

Revision ID: <auto>
Revises: <previous revision>
Create Date: <auto>
"""
from alembic import op
import sqlalchemy as sa

def upgrade() -> None:
    op.add_column(
        "chat",
        sa.Column("tokens_used", sa.Integer(), nullable=False, server_default="0"),
    )

def downgrade() -> None:
    op.drop_column("chat", "tokens_used")
```

Adjust table name (`"chat"`) if it differs in your schema.

----------------------------------------------------------------------
3) Save tokens_used after GPT call
----------------------------------------------------------------------

File: `backend/chat/service.py`

Find the function that:
- calls OpenAI API (GPT),
- saves the assistant message to DB,
- returns the response.

After getting the OpenAI completion, extract token count and save to the `Chat` row:

```python
# after GPT call
tokens = completion.usage.total_tokens  # adjust to actual attribute name

# update chat row
chat = db.query(Chat).filter(Chat.session_id == session_id, Chat.client_id == client_id).first()
if chat:
    chat.tokens_used = tokens
    db.add(chat)
    db.commit()
```

Notes:
- If the response object uses a different attribute for token count, adjust accordingly
  (common: `response.usage.total_tokens`, `completion.usage.total_tokens`).
- If `tokens_used` should be a running total (sum across messages in a session),
  use `chat.tokens_used += tokens` instead of `= tokens`. Decide based on existing logic.
- Do NOT change the return value or the public API of this function.
